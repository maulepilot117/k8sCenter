# Design Context — k8sCenter

## Target Audience
Platform engineers, DevOps teams, and developers with mixed skill levels managing Kubernetes clusters.

## Use Cases
Day-to-day cluster operations, daily monitoring, new deployments, incident response, resource management.

## Brand Personality & Tone
Modern, professional, technical. The interface should feel like a capable power tool — dense where needed, clear always. Confidence-inspiring without being intimidating.

## Design System
- **Themes**: 7 named dark themes (Nexus default, Dracula, Tokyo Night, Catppuccin, Nord, One Dark, Gruvbox)
- **Typography**: Geist Sans (UI) + Geist Mono (data/code)
- **Navigation**: 56px icon rail (replaced 240px sidebar), 8 domain sections with sub-nav tabs
- **Layout**: Islands architecture (Fresh 2.x/Preact), Tailwind v4 utility-only
- **Colors**: CSS custom properties via `[data-theme]` attribute selectors
- **Components**: WizardStepper pattern, ResourceTable, split pane, command palette (Cmd+K)
