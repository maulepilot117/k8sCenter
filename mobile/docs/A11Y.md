# k8sCenter Mobile ‚Äî Accessibility (WCAG 2.2 AA)

PR-5h (M5) brought every M1‚ÄìM4 surface to WCAG 2.2 AA. This document
captures the contract, the helpers added to enforce it, the manual smoke
checklist run before each release, and the pattern reference future
work should follow.

## Patterns to follow when adding new screens

### Status chips / pills

The chip widget owns its `Semantics(container: true, label: ‚Ä¶)`. The
callers do NOT re-wrap. Pattern (see `lib/features/eso/eso_widgets.dart`
`DriftPill`, `lib/features/policy/policy_widgets.dart` `SeverityBadge`):

```dart
class _MyPill extends StatelessWidget {
  final String state;

  @override
  Widget build(BuildContext context) {
    return Semantics(
      container: true,
      label: 'My-domain state: $state',
      excludeSemantics: true, // the visible text label is announced
                              // by the Semantics wrapper, not the inner
                              // Text ‚Äî prevents a doubled read.
      child: Container(/* visuals */, child: Text(state)),
    );
  }
}
```

### Icon-only buttons

Always provide `tooltip:`. Tooltip doubles as the semantics label per
Flutter convention. If the affordance isn't an `IconButton`, wrap in
`Semantics(button: true, label: ‚Ä¶, child: ‚Ä¶)`.

```dart
IconButton(
  tooltip: 'Refresh',
  icon: Icon(Icons.refresh),
  onPressed: _refresh,
)
```

### Decorative icons next to text

Use `MergeSemantics` to collapse the icon-text pair into a single
accessibility node. The icon is excluded so the screen reader announces
only the text node.

```dart
MergeSemantics(
  child: Row(children: [
    ExcludeSemantics(child: Icon(Icons.cloud_outlined)),
    SizedBox(width: 8),
    Text('Cluster status: healthy'),
  ]),
)
```

### Pure decorative icons

Wrap in `ExcludeSemantics`. Most chevrons, status indicators that
duplicate adjacent text, and folder-icon prefixes fall in this category.

```dart
trailing: ExcludeSemantics(child: Icon(Icons.chevron_right)),
```

### Charts (KubeGaugeRing, KubeLineChart)

`KubeGaugeRing` accepts an optional `semanticsLabel` parameter ‚Äî pass a
human-readable summary like `'Sync rate: 87%, healthy'`.

For `KubeLineChart`, wrap the chart container in a `Semantics(label: ‚Ä¶)`
that summarises the chart contents at a high level
(`'Request rate over the last hour: peak 1.2k/s'`). The legend rows
already use `MergeSemantics` internally.

### Filter chips (vulnerabilities list, similar)

Selectable filter affordances need both `button: true` and `selected:`
on the `Semantics` wrapper so screen readers announce the toggle state.

```dart
Semantics(
  button: true,
  selected: isSelected,
  label: '$severity filter',
  child: ChoiceChip(/* visuals */),
)
```

### Sensitive plaintext masking

Secrets screen masks unrevealed values with `‚Ä¢` glyphs. The glyph string
is excluded from semantics (`ExcludeSemantics`) and a `Semantics(label:
'masked value')` wraps the masked branch. Revealed values render their
actual content via the normal Text semantics ‚Äî no override needed.

## Manual smoke checklist (run before each store release)

### iOS (Simulator ‚Üí VoiceOver)

1. Launch the app in VoiceOver mode (`Settings ‚Üí Accessibility ‚Üí
   VoiceOver ‚Üí On`, or `Cmd+F5` on macOS).
2. Login screen:
   - Swipe right through the form ‚Äî order should be username ‚Üí password
     ‚Üí submit ‚Üí OIDC provider 1 ‚Üí OIDC provider 2.
   - Each affordance should announce a recognizable label (no raw
     `image`, no `unlabeled button`).
3. Dashboard:
   - Cards announce as "$count $kind, $tier" (e.g., "47 Pods, healthy").
4. Resource detail (open a Pod):
   - Header: back button announces as "Back, button"; title announces
     as `${podName}, ${namespace}`; status pill announces as
     "Status: Running".
   - Tab bar: each tab announces as `${tabName}, tab, 1 of 5` (or
     similar). Tab order Overview ‚Üí YAML ‚Üí Events ‚Üí Metrics ‚Üí Logs.
5. ESO ExternalSecret detail:
   - Drift pill announces as "Drift state: InSync".
   - Force Sync button announces with a clear action label.

### Android (Emulator ‚Üí TalkBack)

1. Enable TalkBack: `Settings ‚Üí Accessibility ‚Üí TalkBack ‚Üí On`.
2. Repeat steps 2‚Äì5 from the iOS list. Differences:
   - Tab announcements include role + position via TalkBack default.
   - `FLAG_SECURE` is active on Secret detail with revealed values ‚Äî
     verify the app-switcher snapshot is blanked.

### Dynamic type (iOS + Android)

1. iOS: `Settings ‚Üí Display & Brightness ‚Üí Text Size ‚Üí Largest` (or
   `Accessibility ‚Üí Display & Text Size ‚Üí Larger Text` for the wider
   range).
2. Android: `Settings ‚Üí Display ‚Üí Font size ‚Üí Largest`.
3. Walk through login, dashboard, resource list, resource detail. No
   text should be clipped. Critical buttons (Submit, Sign in) must
   remain reachable on phone form factor.

### Reduced motion

1. iOS: `Settings ‚Üí Accessibility ‚Üí Motion ‚Üí Reduce Motion ‚Üí On`.
2. Android: `Settings ‚Üí Accessibility ‚Üí Remove animations ‚Üí On`.
3. Open the cluster picker / wizard sheet ‚Äî transitions should reduce
   to cuts (Flutter handles this automatically when
   `MediaQuery.disableAnimations` is true).

## Deferred to follow-up work

Nothing in M1‚ÄìM4 surfaces was deferred ‚Äî the PR-5h pass covered every
affordance. M5 surfaces (Settings, Onboarding, ESO write modals, OIDC
button) ship with semantics annotations in their own PRs and are not
subject to a separate a11y pass.

Cross-cutting opportunities for M6+:

- **Per-provider OIDC button branding.** The current Sign-in buttons use
  the generic "Sign in with $providerName" label. Per-provider icons
  (Google, Microsoft, Apple) would improve glanceability ‚Äî already
  documented as a follow-up in the M5 plan, not blocked by a11y work.
- **Live region announcements** for state changes (e.g., "Force Sync
  succeeded", "Drift cleared"). Login screen error banners use
  `liveRegion: true`; extending the pattern to other transient
  success/failure surfaces is a natural next step.
- **Locale + RTL.** k8sCenter is English-only today. When localisation
  lands, the existing semantics labels will need to be locale-aware
  (they're currently hard-coded English strings).
